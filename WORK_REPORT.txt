================================================================================
                            PROJECT OVERVIEW
================================================================================

ePair is a full-stack service booking platform consisting of:
- Backend: Laravel PHP Framework (REST API)
- Frontend: Flutter Cross-Platform Application (Android, iOS, Web)

The project aims to provide a comprehensive service booking system where users 
can browse services, book appointments, track bookings, communicate with 
providers, and manage payments.

PROBLEM 3: X509Certificate Compilation Error

  - No google-services.json file present for Android
  - Firebase initialization failed silently, blocking app initialization
  - App crashed/hung during startup sequence
Attempted Solutions:
  ✓ Tested backend API (working correctly)
  ✓ Added Google Maps API key to AndroidManifest.xml
  ✓ Updated backend URL to network IP (192.168.29.100:8000)
  ✓ Added Windows Firewall rule for backend port
  ✓ Multiple backend restarts
Final Solution:
  - Commented out Firebase.initializeApp() for mobile in main.dart
  - Commented out Firebase Messaging initialization calls
  - Kept Firebase active for web builds only
  - App now loads successfully without Firebase configuration
Status: ✓ RESOLVED

PROBLEM 10: Backend Returning HTML Instead of JSON
-----------
Issue: Backend API returning HTML login page instead of JSON data
  - API endpoint returning status 200 but with HTML content
  - App receiving HTML when expecting JSON configuration data
  - Caused parsing errors in Flutter app
Solution:
  - Killed all PHP server processes
  - Cleared Laravel configuration cache (php artisan config:clear)
  - Restarted Laravel development server
  - API now correctly returns JSON responses
Status: ✓ RESOLVED

PROBLEM 11: JAVA_HOME Cached by Gradle Daemon
-----------
Issue: Gradle continued using old invalid JAVA_HOME despite environment update
  - Error: "JAVA_HOME is set to an invalid directory: jdk-17.x.x"
  - Environment variables were correct at system level
  - Gradle daemon cached old environment variables
Solution:
  - Stopped Gradle daemon: .\gradlew --stop
  - Deleted Gradle cache directory (.gradle/)
  - Added org.gradle.java.home to android/local.properties
  - Rebuilt APK successfully
Status: ✓ RESOLVED
